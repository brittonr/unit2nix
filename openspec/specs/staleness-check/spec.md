## Requirements
### Requirement: Build plan includes Cargo.lock hash
The unit2nix CLI SHALL compute a SHA256 hash of the `Cargo.lock` file content and include it as `cargoLockHash` in the top-level JSON output.

#### Scenario: Hash is present in generated JSON
- **WHEN** `unit2nix -o build-plan.json` runs successfully
- **THEN** the output JSON SHALL contain a top-level `cargoLockHash` field with a hex-encoded SHA256 hash string

#### Scenario: Hash changes when Cargo.lock changes
- **WHEN** `Cargo.lock` is modified (e.g., a dependency is added or bumped)
- **THEN** re-running `unit2nix` SHALL produce a different `cargoLockHash` value

### Requirement: Nix consumer validates staleness at eval time
The `buildFromUnitGraph` function SHALL compare the `cargoLockHash` from the JSON with `builtins.hashFile "sha256"` of the workspace's `Cargo.lock`. If they differ, evaluation SHALL fail with an error message that includes the regeneration command.

#### Scenario: Matching hashes pass silently
- **WHEN** `build-plan.json` was generated from the current `Cargo.lock`
- **THEN** `buildFromUnitGraph` SHALL evaluate without error

#### Scenario: Mismatched hashes fail with clear error
- **WHEN** `Cargo.lock` has changed since `build-plan.json` was generated
- **THEN** evaluation SHALL fail with an error containing "build-plan.json is out of date" and "unit2nix" in the message

### Requirement: Staleness check can be disabled
The `buildFromUnitGraph` function SHALL accept a `skipStalenessCheck` parameter (default `false`). When set to `true`, the hash comparison SHALL be skipped.

#### Scenario: Staleness check skipped when opted out
- **WHEN** `buildFromUnitGraph` is called with `skipStalenessCheck = true`
- **THEN** evaluation SHALL succeed even if `cargoLockHash` does not match the current `Cargo.lock`

#### Scenario: Staleness check skipped when hash is absent
- **WHEN** `build-plan.json` does not contain a `cargoLockHash` field (e.g., generated by an older version of unit2nix)
- **THEN** evaluation SHALL succeed without error (backwards compatible)
